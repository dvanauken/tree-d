# Parametric Tree Editor Phase Plan

North star: a professional parametric tree creator/editor for site plans.
Trees must be dimensionally credible, species-based, editable, deterministic,
and suitable for plan, elevation, and perspective presentation.

Live Oak is the first benchmark species. It is not the whole product.

No clip art. No symbol-only shortcuts. No LOD, export pipeline, impostors, or
broad species expansion until the generated tree structure is credible.

## Phase 0: Baseline

- Freeze reference Live Oak cases for young, mature, and old.
- Capture screenshots and model metrics for repeatable comparison.
- Track plan/elevation/perspective needs against the same generated tree model.

Deliverable: stable references for judging whether tree changes improve the
professional site-plan output.

## Phase 1: Parametric Site-Plan Controls

- Keep all dimensions in feet.
- Treat height, spread, DBH, clear trunk, crown form, limb count, branch depth,
  taper, sweep, sag, and foliage density as editable tree parameters.
- Preserve deterministic seeds so trees can be regenerated and adjusted.

Deliverable: parameter changes produce predictable, dimensionally meaningful
tree changes.

## Phase 2: Structural Branch Model

- Move from point-to-point branch paths toward curve-based branch spines.
- Keep the skeleton editable and deterministic.
- Avoid decorative mesh tricks that do not improve the underlying structure.

Deliverable: major limbs read as grown structure, not segmented generated tubes.

## Phase 3: Major Fork Geometry

- Stop treating major forks as intersecting independent tubes.
- Build a real strategy for trunk-to-primary and primary-to-secondary junctions.
- Use simplified treatment only for smaller branch and twig forks.

Deliverable: visible scaffold junctions are credible at presentation scale.

## Phase 4: Live Oak Benchmark Morphology

- Tune low scaffold limbs, broad canopy spread, sweep, sag, asymmetry, and
  old-tree character.
- Make the tree read correctly with leaves off.
- Use Live Oak as the quality gate for the editor's structural system.

Deliverable: a benchmark Live Oak suitable for site-plan presentation.

## Phase 5: Foliage From Structure

- Attach foliage to branch and twig structure.
- Avoid using leaves to hide weak branch geometry.
- Keep foliage editable by density, size, distribution, and species habit.

Deliverable: foliage supports the generated structure and site-plan use case.

## Phase 6: Site-Plan Views

- Derive plan, elevation, and perspective views from the same parametric model.
- Keep figure/grid/scale references dimensionally meaningful.
- Do not use clip art or static symbols as substitutes for generated trees.

Deliverable: the editor produces coherent site-plan views from one tree model.

## Phase 7: Verification

- Run deterministic model checks.
- Compare reference seeds before and after each engine change.
- Do visual review in plan, elevation, and perspective.
- Avoid UI redesign work except when needed for tree editing or verification.

Deliverable: verified progress toward a professional parametric tree editor.
